HttpFetch

HttpFetch: function
  • Defined in fetch/http-fetch.ts:25

HTTP fetch function signature.

This is a function that wraps browser's fetch() function and provides additional functionality. E.g. request interception.

This function returns an OnEvent registrar of event receivers instead of a Promise. This allows to report multiple responses. E.g. when the resource refresh requested and delivered. The request is sent whenever a receiver is registered in this registrar. When the fetch completes the events are exhausted, and callbacks registered with EventInterest.whenDone() method of event interest returned on receiver registration, are notified. When fetch fails for whatever reason, this reason is passed to these callbacks. The fetch can be aborted by losing interest to the response. I.e. by calling EventInterest.off() method.

An instance of HttpFetch is available from bootstrap context.

Type declaration

    • (this: void, input: RequestInfo, init?: RequestInit): OnEvent<[Response]>

    • Parameters

      • this: void
      • input: RequestInfo

        The resource to fetch. This can either an URL string, or a Request object.

      • Optional init: RequestInit

        Custom settings to apply to the request.

      Returns OnEvent<[Response]>

      An OnEvent registrar of response object(s) receivers.
